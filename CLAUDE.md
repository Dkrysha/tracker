# Проект: трекер прогресса (для меня и друга)

Личное приложение для отслеживания знакомств. Общая база на двоих.

## Стек
- Streamlit (интерфейс, Python), venv
- Supabase (PostgreSQL + вход через Google)
- Деплой позже: GitHub → Streamlit Community Cloud (приватное приложение)

## Две таблицы
- day_logs: id, user_email, date, seen (увидел), approaches (подошёл)
- contacts: id, user_email, name, source (улица/соцсети/сайт),
  stage (контакт/свидание/закрытие), notes (свободные заметки, общие, в статистику НЕ идут)

## Три вкладки
- «Сегодня»: два счётчика (+1 увидел / +1 подход) → запись в day_logs
- «Карточки»: создать/список/сменить стадию контакта + поле заметок
- «Статистика»: воронка увидел→подошёл→контакт→свидание→закрытие + сравнение с другом

## Логика воронки
- Верх (увидел, подошёл) — только улица, из day_logs
- Низ (контакт→свидание→закрытие) — счёт карточек по стадии
- Конверсию подход→контакт считать только по source=улица; ниже можно фильтровать по каналу

## Принцип
Максимально просто. Запись должна занимать секунды, иначе её перестанут вести.

## Решение по входу (зафиксировано)
- Вход через Google НЕ используем — ни в коде, ни при деплое.
- Кто вносит запись определяет селектор внутри приложения. Это постоянное решение.
  Поле user_email хранит просто метку человека (Danylo / Pavlo), а не настоящий
  e-mail — это нормально, переезжать на реальные почты не нужно.
- Защита доступа: один общий пароль на входе (из secrets.toml), добавим при деплое.
